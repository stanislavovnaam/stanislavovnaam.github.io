Правила Наименования Файлов и Папок
- Файлы: имя-файла.html
- Папка: имя-папки

HTML Правила:
- используем только классы для стилей
- id только для кода
- классы создаем либо по принципу иерархии:
    - class="banner"
    - class="banner-box"
- либо по принципу переиспользуемых шаблонов:
     - class="title"
     - class="subtitle
- используем теги по стандартной семантике
     - один h1 на странице, h3 после h2, <p> для абзацев и т.д.
- используем комментарии
    - для визуального разделения кода
    - для сложных моментов

CSS Правила:
- обнуляем стили в файле base.css
- для каждого типа секции отдельный .css файл
- цветовая тема в отдельном файле theme.css
- используем только классы
- не забываем про комплексные селекторы:
    - .класс1 .класс2 {} для настройки 2-го в 1-ом
    - .класс1, .класс2 {} для настройки 2-го и 1-го
    - и т.д.

JS Правила:
- весь основной код в main.js
- переменные:
    - используют let
    - названы по правилу верблюда: крутаяПеременнаяОдин
    - названы по назначению
- функции:
    - названы по правилу верблюда: крутаяФункция

Адаптация под устройства:
- используем динамические единицы измерения
- создаем дизайн для мобильных устройств в первую очередь
     - используем тег meta с атрибутом viewport
- используем только flexbox
- используем css переменные для шкалы масштаба

Единицы Измерения:
- размер шрифта: rem
- отступы: rem или %
- высота / ширина: %, min(), max(), clamp() auto
- радиус границы: px или rem
- высота строки: без единицы измерения
- медиа запросы: em или rem
